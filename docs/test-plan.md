# テスト計画書: 文字数しりとり

## 概要

本ドキュメントは「文字数しりとり」アプリケーションのテスト計画を記述する。

## テスト方針

### テスト対象の選定基準

| 観点       | 判断     | 内容                                                       |
| ---------- | -------- | ---------------------------------------------------------- |
| 単体テスト | **実施** | 純粋関数として分離されているロジックをテスト。Vitest使用。 |
| 結合テスト | **実施** | ページ数も少なく、テスト項目も少ないため、手動テスト。     |

### テストフレームワーク

- **Vitest** を使用
- テスト専用設定（`vitest.config.js`）を分離し、Cloudflare プラグインとの競合を回避

## 単体テスト

### 1. `wordUtils.js` — しりとりの中核ロジック

#### 1.1 `getHeadChar(history)` — 次の頭文字を決定する関数

前の単語の末尾文字から、次に入力すべき頭文字を返す。

| #   | テストケース           | 入力                              | 期待値 | 検証するルール               |
| --- | ---------------------- | --------------------------------- | ------ | ---------------------------- |
| 1   | 履歴が空               | `[]`                              | `"り"` | 初期値は「り」               |
| 2   | 通常文字で終わる       | `[{word:"りんご"}]`               | `"ご"` | 基本の末尾取得               |
| 3   | 小文字「ょ」で終わる   | `[{word:"きんぎょ"}]`             | `"よ"` | 小文字→大文字変換            |
| 4   | 小文字「っ」で終わる   | `[{word:"りらっ"}]`               | `"つ"` | 小文字→大文字変換            |
| 5   | 長音符「ー」で終わる   | `[{word:"こーひー"}]`             | `"ひ"` | 長音符スキップ               |
| 6   | 複数履歴がある         | `[{word:"りんご"},{word:"ごま"}]` | `"ま"` | 最後の単語のみ参照           |
| 7   | 長音符＋小文字の組合せ | `[{word:"りゅー"}]`               | `"ゆ"` | 長音符スキップ後に小文字変換 |

#### 1.2 `validateWord(word, cardConfig, history)` — 単語バリデーション

入力された単語が全ルール（文字種・文字数条件・頭文字一致）を満たすか検証する。
各条件タイプについて境界値を網羅的にテストする。

| #   | テストケース                      | 期待           | 検証するルール                   |
| --- | --------------------------------- | -------------- | -------------------------------- |
| 1   | 空文字                            | `valid: false` | 空入力の拒否                     |
| 2   | カタカナを含む                    | `valid: false` | ひらがな限定チェック             |
| 3   | 英字を含む                        | `valid: false` | ひらがな限定チェック             |
| 4   | `exact(3)`: 3文字（一致）         | `valid: true`  | ちょうどN文字（正常系）          |
| 5   | `exact(3)`: 2文字（1少ない）      | `valid: false` | ちょうどN文字（境界値-1）        |
| 6   | `exact(3)`: 4文字（1多い）        | `valid: false` | ちょうどN文字（境界値+1）        |
| 7   | `min(3)`: 3文字（境界値ちょうど） | `valid: true`  | N文字以上（境界値）              |
| 8   | `min(3)`: 4文字（境界値+1）       | `valid: true`  | N文字以上（境界値超え）          |
| 9   | `min(3)`: 2文字（境界値-1）       | `valid: false` | N文字以上（境界値未満）          |
| 10  | `range(2-4)`: 2文字（下限）       | `valid: true`  | N〜M文字（下限境界値）           |
| 11  | `range(2-4)`: 4文字（上限）       | `valid: true`  | N〜M文字（上限境界値）           |
| 12  | `range(2-4)`: 3文字（範囲内）     | `valid: true`  | N〜M文字（中央値）               |
| 13  | `range(2-4)`: 1文字（下限-1）     | `valid: false` | N〜M文字（下限境界値外）         |
| 14  | `range(2-4)`: 5文字（上限+1）     | `valid: false` | N〜M文字（上限境界値外）         |
| 15  | 頭文字不一致                      | `valid: false` | しりとり接続チェック             |
| 16  | 長音符を含む単語                  | `valid: true`  | 長音符がバリデーションを妨げない |

#### 1.3 `isNEnding(word)` — 「ん」終わり判定

ゲーム終了条件の一つである「ん」で終わる単語を判定する。コンポーネント（`JudgeButtons.jsx`）から抽出。

| #   | テストケース       | 期待    | 検証するルール |
| --- | ------------------ | ------- | -------------- |
| 1   | 「ん」で終わる     | `true`  | 基本の終了判定 |
| 2   | 「ん」で終わらない | `false` | 非該当の確認   |

### 2. `formatTime.js` — 時間表示フォーマット

#### 2.1 `formatTime(seconds)` — 秒数を MM:SS 形式に変換

コンポーネント（`PlayerTimer.jsx`, `ResultTimer.jsx`）に重複していたロジックを抽出。分の桁上がりの境界値を中心にテストする。

| #   | テストケース                  | 入力  | 期待値    | 検証するルール         |
| --- | ----------------------------- | ----- | --------- | ---------------------- |
| 1   | 0秒                           | `0`   | `"00:00"` | ゼロ値                 |
| 2   | 59秒（分の境界直前）          | `59`  | `"00:59"` | 分に繰り上がらない上限 |
| 3   | 60秒（ちょうど1分）           | `60`  | `"01:00"` | 分の境界値             |
| 4   | 61秒（1分の境界直後）         | `61`  | `"01:01"` | 分の境界値+1           |
| 5   | 300秒（デフォルトゲーム時間） | `300` | `"05:00"` | 実際のゲーム設定値     |
| 6   | 599秒（2桁分の境界直前）      | `599` | `"09:59"` | 分が1桁の上限          |
| 7   | 600秒（2桁分）                | `600` | `"10:00"` | 分が2桁になる境界値    |

### 3. `gameUtils.js` — カードドロー

#### 3.1 `drawCard(cards)` — 有効カードからランダムに1枚引く

| #   | テストケース           | 検証内容                                                 |
| --- | ---------------------- | -------------------------------------------------------- |
| 1   | 有効カードが1枚のみ    | `Math.random` モックでそのカードが返ることを確認         |
| 2   | 無効カードは選ばれない | 50回繰り返し実行し、無効カードが一度も返らないことを確認 |
| 3   | 全カード有効           | 返り値が `CARD_ORDER` に含まれることを確認               |

### 4. `settingsUtils.js` — 設定の永続化

#### 4.1 `loadSettings()` — localStorage から設定を読み込む

| #   | テストケース | 期待値             | 検証内容                   |
| --- | ------------ | ------------------ | -------------------------- |
| 1   | データなし   | `DEFAULT_SETTINGS` | 初回起動時のフォールバック |
| 2   | 有効な JSON  | パース結果         | 正常な保存データの復元     |
| 3   | 壊れた JSON  | `DEFAULT_SETTINGS` | データ破損時の耐障害性     |

## 結合テスト

以下に基づき手動で実施する。

### タイトル画面

- ゲームスタートボタンを押下すると、設定画面に遷移する

### 設定画面

- プレイヤー名を空文字にしたとき、デフォルトのプレイヤー名が入力される
- プレイヤー名を8文字以上にしたとき、7文字以降の文字は削除される
- 先攻プレイヤーのプルダウンに、プレイヤーAとプレイヤーBが表示される
- ゲーム時間のプルダウンに、1～30が表示される
- 有効なカードが1つのとき、そのカードを無効にできない
- ちょうどを選択したとき、文字数はカードに対応したものになり、変更できない
- 範囲を選択したとき、2～3文字と表示される
- 範囲を選択したとき、最小値は2～12、最大値は3～13で選択できる
- 範囲を選択したとき、2～3の状態から3～3とすると、自動で3～4になる
- 範囲を選択したとき、3～4の状態から3～3とすると、自動で2～3になる
- 以上を選択したとき、2～13から選択できる
- デフォルトに戻すを押下すると、設定がデフォルトに戻る
- ゲームスタートを押下すると、ゲーム画面に遷移し、設定値がゲームに反映されている
- 設定をデフォルトから変更してゲームスタートして設定画面に戻ったときに、変更後の設定が設定されている

### ゲーム画面 — カウントダウン

- 画面表示時に「3」→「2」→「1」→「スタート!」と1秒ずつ表示される
- カウントダウン中は単語入力欄と確定ボタンが無効になっている
- カウントダウン終了後、カードが1枚引かれプレイ状態になる

### ゲーム画面 — プレイ中

- 現在のプレイヤーのタイマーが青背景・青枠で強調される
- 現在のプレイヤーのタイマーのみカウントダウンされる
- カード・頭文字・文字数条件が表示される
- 初回ターンの頭文字が「り」になっている
- 空文字で確定するとエラーメッセージが表示される
- カタカナや英字を含む入力で確定するとエラーメッセージが表示される
- 文字数条件を満たさない入力で確定するとエラーメッセージが表示される（ちょうど、範囲、以上のそれぞれで確認）
- 頭文字が合わない入力で確定するとエラーメッセージが表示される
- 条件を満たす単語で確定すると、入力した単語が表示され判定ボタンが現れる
- Enterキーで確定できる

### ゲーム画面 — 判定

- 「有効」を押下すると、ターンが交代し新しいカードが引かれる
- ターン交代後、前の単語の末尾から正しい頭文字が設定される
- 「無効」を押下すると、入力がクリアされ同じプレイヤーが再入力できる
- 「審議」を押下すると、審議モーダルが表示される

### ゲーム画面 — 審議モーダル

- 入力された単語が大きく表示される
- 「Googleで検索」ボタンを押下すると、新しいタブでGoogle検索が開く
- モーダル内の判定ボタンに「審議」ボタンが表示されない（有効・無効のみ）
- モーダル内で「有効」「無効」を押下した場合、通常の判定と同じ動作をする
- ×ボタンでモーダルを閉じると、判定画面に戻る

### ゲーム画面 — ゲーム終了

- 「ん」で終わる単語を「有効」にした場合、「「ん」で終わった！」と表示される
- プレイヤーの残り時間が0になった場合、「時間切れ！」と表示される
- ゲーム終了後、3秒で結果画面に自動遷移する

### ゲーム画面 — 設定モーダル

- 設定アイコンを押下するとメニューモーダルが開く
- 「リプレイ」を押下すると、現在の設定でゲームが最初からやり直される
- 「設定変更」を押下すると、設定画面に遷移する
- 「タイトルへ」を押下すると、タイトル画面に遷移する

### ゲーム画面 — 履歴

- 履歴セクションを開閉できる
- 回答した単語がプレイヤー名とともに履歴に表示される
- 新しい単語が上に表示される（降順）

### 結果画面

- タイムアウトで終了した場合、時間切れになったプレイヤーの相手が勝者として表示される
- 「ん」で終わる単語を出したプレイヤーの相手が勝者として表示される
- 勝者のプレイヤー名が「〇〇の勝利！」と表示される
- 両プレイヤーの残り時間がMM:SS形式で表示される
- ゲーム中の履歴が表示される
- 「リプレイ」を押下すると、現在の設定でゲームが開始される
- 「設定変更」を押下すると、設定画面に遷移する
- 「タイトルへ」を押下すると、タイトル画面に遷移する
- URLに直接アクセスした場合、タイトル画面にリダイレクトされる
