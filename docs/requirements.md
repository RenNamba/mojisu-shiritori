# 文字数しりとり 要件定義書

バージョン 2.0　作成日：2026年3月

## 1. プロジェクト概要

本プロジェクトは、トランプのカードを使って文字数制限を加えたしりとりゲームを React で実装し、ポートフォリオとして公開することを目的とする。

### 1.1 成果物

- アプリ URL（Cloudflare Pages にデプロイ）
- GitHub リポジトリ
- README
- 設計書・テスト計画書

## 2. ゲームルール

### 2.1 基本ルール

- 2人対戦で同じ画面を見てプレイする
- 基本的にはしりとりのルールに準拠する
- 単語回答時は、ランダムにカードを引き、引いたカードの文字数条件に従って単語を回答する必要がある

### 2.2 負け条件

- タイマーが0になった場合
- 「ん」で終わる単語を回答し、相手プレイヤーが有効ボタンを押した場合

### 2.3 カードのルール

デフォルト設定は以下の通り。設定画面で変更可能。

| カード | デフォルト ON/OFF | デフォルト条件   | 備考                 |
| ------ | ----------------- | ---------------- | -------------------- |
| 2〜10  | ON                | 出目と同じ文字数 | 例：7を引いたら7文字 |
| J      | ON                | 11文字以上       | 設定で変更可能       |
| Q      | OFF               | 12文字以上       | 設定で変更可能       |
| K      | OFF               | 13文字以上       | 設定で変更可能       |
| A      | 除外              | なし             | 設定画面に表示しない |

### 2.4 1ターンの流れ

1. ターン開始：有効カードからランダムに1枚引かれ、文字数条件が表示される
2. 回答者が単語を口頭で宣言し、入力欄に入力して確定ボタンを押す
3. 確定ボタンを押した瞬間にタイマー一時停止。相手プレイヤーが「有効」「無効」「審議」のいずれかを選択する
   - 有効 → 履歴に追加・次のプレイヤーのターンへ（「ん」で終わる単語の場合はゲーム終了）
   - 無効 → 回答者が同じカードで再回答
   - 審議 → 審議モーダルが開く（詳細は審議フロー参照）

### 2.5 審議フロー

1. 相手プレイヤーが「審議」ボタンを押す
2. 審議モーダルが表示される（回答した単語が自動でセットされた状態）
3. 相手プレイヤーが検索ボタンで外部ブラウザ（Google）を開いて確認
4. 相手プレイヤーが「有効」または「無効」を選択
   - 有効 → 履歴に追加・ゲーム再開・次のプレイヤーのターンへ（「ん」で終わる場合はゲーム終了）
   - 無効 → 回答者が同じカードで再回答

## 4. 画面詳細

### 4.1 タイトル画面

- タイトル表示
- ゲームスタートボタン → 設定画面へ遷移

### 4.2 設定画面

#### プレイヤー設定

- プレイヤー名入力（2名）　
  - デフォルト：プレイヤーA・プレイヤーB　
  - 最大7文字で8文字以上にすると7文字目以降は自動削除
  - 空文字の場合はデフォルト名が自動入力
- 先攻後攻選択　
  - デフォルト：プレイヤーAが先攻

#### ゲーム設定

- ゲーム時間設定（各プレイヤーのタイマー）　
  - 単位：分　
  - 範囲：1〜30分　
  - デフォルト：5分

#### カード文字数条件設定

- 2〜Kの各カードに対して個別に設定可能
- 各カードにデッキに含める / 含めないの ON/OFF トグルあり
- OFF にしても条件設定はリセットされない（設定値を保持）
- ON のカードが残り1件になった場合、そのトグルは非活性にする（全 OFF を防止）
- OFF のカードは条件タイプ・値の入力欄を非活性にする
- デフォルト ON：2〜J　デフォルト OFF：Q・K　A は除外（表示しない）
- 条件タイプはプルダウンで選択：ちょうど / 範囲 / 以上

| 条件タイプ | 入力欄                                      | 例                        |
| ---------- | ------------------------------------------- | ------------------------- |
| ちょうど   | カードの数字が自動セット・非活性            | 7のカード → 7文字（固定） |
| 範囲       | 最小・最大をプルダウン（2〜13）で選択・連動 | 8〜10文字                 |
| 以上       | プルダウン（2〜13）で選択                   | 11文字以上                |

#### その他

- 設定は localStorage に保存・次回起動時に復元
- 「デフォルトに戻す」ボタン → 全設定をデフォルト値にリセット
- 「ゲーム開始」ボタン → 設定を localStorage に保存してゲーム画面へ遷移

### 4.3 ゲーム画面

#### レイアウト（上から）

| ゾーン       | 内容                                                       |
| ------------ | ---------------------------------------------------------- |
| 上部         | 各プレイヤー名とタイマー。現在のプレイヤーをハイライト表示 |
| プレイエリア | 回答状況に応じて2パターン切り替え（下記参照）              |
| 入力欄       | 単語入力欄（ひらがなで入力の注意書きあり）+ 確定ボタン     |
| 履歴         | 展開ボタンを押すと表示。スクロール表示                     |

#### ゲーム状態一覧

| 状態         | 説明                                    | タイマー       |
| ------------ | --------------------------------------- | -------------- |
| countdown    | 3・2・1カウントダウン中                 | 両者ストップ   |
| playing      | 回答中                                  | 回答者のみ減る |
| answered     | 回答後（有効・無効・審議待ち）          | 両者ストップ   |
| deliberation | 審議中（審議モーダル表示）              | 両者ストップ   |
| setting      | 設定モーダル表示中                      | 両者ストップ   |
| finished     | ゲーム終了モーダル表示・3秒後結果画面へ | 両者ストップ   |

#### プレイエリア：回答中（playing）

- 引いたカード表示
- 頭文字表示（最初のターンは「り」固定、以降は前の単語の末尾文字）
- 文字数条件表示
- 入力欄・確定ボタンは活性

#### プレイエリア：回答後（answered / deliberation）

- 引いたカード表示
- 頭文字・文字数条件表示
- 回答した単語表示
- 有効・無効・審議ボタン
- 入力欄・確定ボタンは非活性

#### 入力バリデーション

上から順にチェックし、最初に引っかかったエラーを表示する。

| 順番 | チェック                   | 条件                            | エラーメッセージ                       |
| ---- | -------------------------- | ------------------------------- | -------------------------------------- |
| 1    | 空文字チェック             | 入力文字数 === 0                | 「文字を入力してください」             |
| 2    | ひらがなチェック           | ひらがな以外を含む              | 「ひらがなで入力してください」         |
| 3    | 文字数チェック（ちょうど） | 入力文字数 !== 指定文字数       | 「〇文字で入力してください」           |
| 3    | 文字数チェック（範囲）     | 入力文字数が範囲外              | 「〇〜△文字で入力してください」        |
| 3    | 文字数チェック（以上）     | 入力文字数 < 指定文字数         | 「〇文字以上で入力してください」       |
| 4    | 頭文字チェック             | 先頭文字 !== 前の単語の末尾文字 | 「〇から始まる単語を入力してください」 |

#### 頭文字の取得ルール

| ケース                       | 処理                                                |
| ---------------------------- | --------------------------------------------------- |
| 履歴が空（最初のターン）     | 「り」固定                                          |
| 通常文字・濁点・半濁点       | そのまま末尾の1文字を使用                           |
| 小文字（ぁぃぅぇぉっゃゅょ） | 大文字に変換（例：ょ → よ）                         |
| 長音符（ー）                 | 1文字前の文字を使用（その文字にも上記ルールを適用） |

### 4.4 審議モーダル

- 回答した単語が自動でセットされた状態で開く
- 検索ボタン → `https://www.google.com/search?q=単語` を新しいタブで開く
- 有効ボタン → ゲーム再開・次のプレイヤーのターンへ
- 無効ボタン → 回答者のターンに戻る（同じカードを使用）
- 審議モーダル内では「審議」ボタンは表示しない（有効・無効のみ）
- 履歴表示ゾーン（デフォルト非表示・トグルで展開・スクロール表示）

### 4.5 設定モーダル

- 開いている間はタイマー停止
- リプレイボタン → 同じ設定でカウントダウンから再スタート
- 設定変更ボタン → 設定画面へ遷移
- タイトルボタン → タイトル画面へ遷移

### 4.6 カウントダウンモーダル

- ゲーム画面遷移時に自動表示
- 3 → 2 → 1 → スタート! の順に1秒ずつ表示
- スタート! 表示後にモーダルが閉じて playing 状態へ

### 4.7 ゲーム終了モーダル

- どちらかのタイマーが0になった、または「ん」で終わる単語が有効になった瞬間に表示
- 終了理由に応じてメッセージを出し分ける
  - タイマー0 →「時間切れ！」
  - 「ん」で終わった →「「ん」で終わった！」
- 3秒後に自動で結果画面へ遷移

### 4.8 結果画面

- 「〇〇の勝利！」テキスト表示
- 各プレイヤーの残り時間（MM:SS 形式）
- 履歴一覧（有効になった単語のみ・回答者名と単語・スクロール表示）
- リプレイボタン → 同じ設定でカウントダウンから再スタート
- 設定変更ボタン → 設定画面へ遷移
- タイトルボタン → タイトル画面へ遷移
- URL に直接アクセスした場合はタイトル画面にリダイレクト

## 5. データ設計

### 5.1 設定データ（localStorage）

デフォルト値を定数（`DEFAULT_SETTINGS`）として定義し、localStorage に保存・復元する。「デフォルトに戻す」ボタンで `DEFAULT_SETTINGS` に戻す。

| キー                     | 型             | デフォルト値  | 備考                                 |
| ------------------------ | -------------- | ------------- | ------------------------------------ |
| players.A.name           | string         | "プレイヤーA" | プレイヤーAの名前（最大7文字）       |
| players.B.name           | string         | "プレイヤーB" | プレイヤーBの名前（最大7文字）       |
| firstPlayer              | "A" \| "B"     | "A"           | 先攻プレイヤー                       |
| gameTime                 | number         | 300           | ゲーム時間（秒）。1〜30分 = 60〜1800 |
| cards[key].enabled       | boolean        | true / false  | key は "c2"〜"cK"。デッキに含めるか  |
| cards[key].conditionType | string         | "exact" 等    | "exact" \| "range" \| "min"          |
| cards[key].value         | number \| null | カードの数字  | exact・min 時に使用                  |
| cards[key].min           | number \| null | null          | range 時の最小文字数                 |
| cards[key].max           | number \| null | null          | range 時の最大文字数                 |

### 5.2 カードマッピング定数

| 定数名             | 内容                                                   |
| ------------------ | ------------------------------------------------------ |
| CARD_EXACT_VALUES  | カード名 → exact 時のデフォルト値（"c2":2 〜 "cK":13） |
| CARD_ORDER         | カードの表示順（["c2","c3",...,"c10","cJ","cQ","cK"]） |
| CARD_DISPLAY_NAMES | カード名 → 表示名（"c2":"2", "cJ":"J" 等）             |

### 5.3 ゲーム状態データ（React の state）

| キー          | 型             | 初期値           | 備考                                                               |
| ------------- | -------------- | ---------------- | ------------------------------------------------------------------ |
| gameStatus    | string         | "countdown"      | countdown / playing / answered / deliberation / setting / finished |
| currentPlayer | "A" \| "B"     | firstPlayer の値 | 現在のターンのプレイヤー                                           |
| currentCard   | string \| null | null             | 引いたカード（"c2"〜"cK"）                                         |
| timers.A      | number         | gameTime         | プレイヤーAの残り秒数                                              |
| timers.B      | number         | gameTime         | プレイヤーBの残り秒数                                              |
| currentWord   | string         | ""               | 入力欄の現在の値・回答後も保持                                     |
| history       | array          | []               | { id, player, word } の配列。有効時のみ追加                        |
| endReason     | string \| null | null             | "timeout" \| "n-ending" \| null                                    |

### 6.2 状態管理方針

- 設定データ（settings）は SettingsPage で管理・localStorage に保存・ゲーム開始時に GamePage で読み込む
- ゲーム状態データはすべて GamePage の `useState` で一元管理する
- 子コンポーネントには props で必要なデータのみ渡す
- 子 → 親へのデータ受け渡しは `useState` の set 関数を props で渡す形で対応
